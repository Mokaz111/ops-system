## 2. 平台级组件部署

### 2.1 基础设施

| 组件 | 部署方式 | 规格 | 用途 |
|------|---------|------|------|
| PostgreSQL | Operator/手动 | 主从 3 节点 | 元数据存储 |
| Redis | Operator | 集群 3 节点 | 缓存、会话 |
| etcd | K8s 内置 | K8s 管理 | 配置存储 |

### 2.2 API 服务

```yaml
# api-server deployment
api-server:
  replicas: 3
  resources:
    requests:
      cpu: "500m"
      memory: "512Mi"
    limits:
      cpu: "2000m"
      memory: "2Gi"
  
  # 健康检查
  livenessProbe:
    httpGet:
      path: /health
      port: 8080
    initialDelaySeconds: 30
    periodSeconds: 10
  
  readinessProbe:
    httpGet:
      path: /ready
      port: 8080
    initialDelaySeconds: 10
    periodSeconds: 5
```

```yaml

  replicas: 3
```

### 2.4 全局 VMCluster

```yaml
# 全局共享 VMCluster (victoria-metrics-k8s-stack)
vm-global:
  enabled: true
  name: "vm-global-shared"
  
  # VMCluster 配置
  vmcluster:
    spec:
      retentionPeriod: "30d"
      replicationFactor: 2
      
      vmstorage:
        replicaCount: 3
        resources:
          requests:
            cpu: "2"
            memory: "4Gi"
          limits:
            cpu: "4"
            memory: "8Gi"
        storage:
          volumeClaimTemplate:
            spec:
              storageClassName: "fast"
              resources:
                requests:
                  storage: "500Gi"
      
      vminsert:
        replicaCount: 2
        resources:
          requests:
            cpu: "500m"
            memory: "512Mi"
          limits:
            cpu: "1000m"
            memory: "1Gi"
      
      vmselect:
        replicaCount: 2
        resources:
          requests:
            cpu: "500m"
            memory: "512Mi"
          limits:
            cpu: "1000m"
            memory: "1Gi"
  
  # 全局 vmauth (多租户路由)
  vmauth:
    enabled: true
    spec:
      staticRoutes:
        - src_paths: ["/insert/*"]
          backend: "http://vm-global-shared-vmcluster-vminsert:8480"
        - src_paths: ["/select/*"]
          backend: "http://vm-global-shared-vmcluster-vmselect:8481"
```

## 3. 租户级组件部署

### 3.1 共享版部署

```
共享版租户部署流程:
┌─────────────────────────────────────────────────────────────┐
│ 1. 创建 Namespace: tenant-vmuser-xxx                       │
│ 2. 配置 ResourceQuota                                       │
│ 3. 部署 VictoriaLogs (独立实例)                             │
│ 4. 部署 n9e-edge                                            │
```

## 4. 网络设计

### 4.1 网络策略

```yaml
# 平台 Namespace 网络策略
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: platform-ns-policy
  namespace: platform
spec:
  podSelector: {}
  policyTypes:
    - Ingress
    - Egress
  ingress:
    # 允许 API Server 被访问
    - from:
        - namespaceSelector:
            matchLabels:
              name: ingress-nginx
      ports:
        - protocol: TCP
          port: 8080
    # 允许 N9E Server 访问
    - from:
        - namespaceSelector: {}
      ports:
        - protocol: TCP
          port: 18000
  egress:
    # 允许访问 PostgreSQL
    - to:
        - namespaceSelector:
            matchLabels:
              name: platform
          podSelector:
            matchLabels:
              app: postgresql
      ports:
        - protocol: TCP
          port: 5432
```

```yaml
apiIngress:
```

## 5. 存储设计

### 5.1 存储类

| 存储类 | 用途 | 特性 |
|--------|------|------|
| `fast` (SSD) | VM 存储、数据库 | 高 IOPS、低延迟 |
| `standard` (HDD) | 日志存储 | 大容量、低成本 |
| `cephfs` | 共享存储 | 跨节点共享 |

### 5.2 PVC 规划

| 组件 | 存储类 | 默认大小 | 最大大小 | 扩容方式 |
|------|--------|---------|---------|---------|
| VMSingle | fast | 50Gi | 500Gi | PVC 扩容 |
| VMStorage | fast | 200Gi | 2Ti | PVC 扩容 |
| VictoriaLogs | standard | 50Gi | 1Ti | PVC 扩容 |
| Grafana | standard | 10Gi | 100Gi | PVC 扩容 |
| PostgreSQL | fast | 50Gi | 500Gi | PVC 扩容 |
| Redis | fast | 10Gi | 50Gi | PVC 扩容 |

## 6. 高可用设计

### 6.1 组件高可用

| 组件 | 副本数 | 负载均衡 | 故障转移 |
|------|--------|---------|---------|
| API Server | 3 | K8s Service | 自动 |
| N9E Server | 3 | K8s Service | 自动 |
| VMCluster | 3+2+2 | DNS/Service | 自动 |
| VMSingle | 1+1 (冷备) | - | 手动 |
| VictoriaLogs | 1+1 | - | 手动 |
| Grafana | 1 | Session | 手动 |

### 6.2 故障处理

| 场景 | 检测方式 | 处理方式 |
|------|---------|---------|
| Pod 故障 | K8s Liveness | 自动重启 |
| 节点故障 | K8s Node Controller | Pod 迁移 |
| 存储故障 | Storage Operator | 告警通知 |
| 服务不可用 | Health Check | 切换流量 |
| 数据丢失 | 备份恢复 | 恢复数据 |
