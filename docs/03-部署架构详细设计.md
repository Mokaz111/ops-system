# 部署架构详细设计（现网对齐版）

## 1. 组件关系

平台由以下组件构成：

- 前端控制台（静态站点）
- 后端 API 服务（Go）
- PostgreSQL（业务数据 + 审计数据）
- Redis（幂等键存储）
- Kubernetes（承载被管理的监控组件）
- 外部系统：Grafana、N9E

## 2. 后端部署要求

- 提供数据库连接配置
- 提供 JWT 密钥
- 可选提供 K8s kubeconfig 或 in-cluster 配置
- 可选提供 Redis 地址（用于幂等）

若 Redis 不可用：

- 平台扩容仍可执行
- 幂等能力退化，日志会输出告警

## 3. Kubernetes 操作边界

后端通过封装层执行：

- Deployment 副本调整
- PVC 扩容
- VMCluster CR `spec` patch

平台级扩容只操作注册目标，不支持任意资源自由输入。

## 4. 运行时健康检查

- `/health`：进程健康
- `/api/v1/health`：API 健康
- `/api/v1/health/db`：数据库连通性

## 5. 环境建议

- 将数据库密码等敏感配置通过环境变量注入
- 对 admin 接口启用严格访问控制
- 开启后端日志采集，保留平台扩容审计链路
