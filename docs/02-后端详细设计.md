# 后端详细设计（现网对齐版）

## 1. 技术栈

- Go + Gin
- GORM + PostgreSQL（主存储 + 审计）
- Redis（幂等键 + JWT 黑名单，可选）
- client-go + Helm SDK（K8s/Helm）
- Zap 结构化日志
- Viper 配置

## 2. 分层结构

```
backend/internal
├── handler        HTTP 层：参数绑定、错误码映射、租户 / 角色断言
├── service        业务规则、跨组件编排、外部调用
├── repository     GORM 持久化、查询 builder、租户 scope 注入
├── model          GORM 模型 + 索引声明 + BeforeCreate
├── middleware     JWT/RateLimit/CORS/RequestLogger/RequireRole
├── idempotency    幂等存储抽象 + Redis 实现
├── integration    渲染器 + applier（k8s/grafana 子下发器） + resolver
├── k8s            client-go 包装 + 多集群 client 缓存（线程安全）
├── helm           Helm Action 包装
├── grafana        REST 客户端 + 多 host 配置注入
├── n9e            N9E 客户端
├── vm             VMOperator CR 操作
├── notify         通知投递 stub
└── server         路由注册（NewRouter）
```

handler 不直接 import GORM；service 不直接处理 HTTP；repository 不感知用户身份。

## 3. 路由与权限

### 3.1 三层鉴权

| 层 | 中间件 | 范围 |
|---|---|---|
| 公共 | — | `/health`、`/api/v1/health[/db]`、`/api/v1/auth/login`、`/api/v1/users/bootstrap` |
| 已登录 | `JWTAuth(secret)` | 全部 GET 接口 + 个人资料 PUT |
| 管理员 | `JWTAuth + RequireRole("admin")` | 所有写操作 + 平台扩容 + 集群/Grafana 主机/模板/指标/日志实例 CRUD |

JWT 校验内部会查 Redis 黑名单（`jwt:blacklist:<jti>`），命中即 401。Logout 把当前 token 的 jti 写入黑名单（TTL = 剩余有效期）。

## 4. 关键业务规则

### 4.1 多租户访问隔离

- repository 层提供 `TenantScope(tenantID *uuid.UUID)`，nil 表示 admin（不过滤）
- handler 通过 `resolveTenantScope(c, userSvc)` 决定传入 nil 还是用户租户
- 写操作额外用 `assertTenantAccess` 在拿到对象后再校验一次

### 4.2 实例扩缩容（策略 A）

- `shared` / `dedicated_cluster`：所有实例级 scale 直接拒绝（落 `scale_events` method=`rejected`）
- `dedicated_single`：只允许 `vertical`（cpu/memory）+ `storage`（PVC 扩容）
- `ScaleService` 内部用 `sync.Map` + `mu.Lock` 串行同 `instance_id` 的并发请求，防止 helm/CR 操作交叉
- 每次 scale 写 `scale_events`，无论成功/失败，包含 `method` ∈ `cr_patch`、`helm_upgrade`、`k8s_native`、`rejected`

### 4.3 平台级扩容

- 仅注册目标（`PlatformScaleService.ListVMClusterTargets`），不允许任意 `namespace/name` 输入
- 流程：`dry_run=true` 生成 `spec_patch` → 二次提交 `dry_run=false + Idempotency-Key` → CR `MergePatch`
- 全部操作落 `platform_scale_audits`：操作者、IP、目标、状态（success/failed/replayed）、`spec_patch`、错误信息

### 4.4 共享集群初始化

- 接口：`POST /platform/scaling/bootstrap/shared/init`
- Chart 固定 `vm/victoria-metrics-k8s-stack`，默认 `monitoring/vm-shared-stack`
- 支持 `dry_run`；正式执行用 Helm `install or upgrade`
- 同样写入 `platform_scale_audits`

### 4.5 接入中心

- **模板软删 + name 活跃唯一**：`uk_integration_tpl_name_active where deleted_at IS NULL`，下架后可同名重建
- **版本约束**：
  - `(template_id, version)` 唯一
  - 删除版本时若被任何活跃安装引用 → `409 ErrIntegrationVersionInUse`
  - 模板下最后一个版本不允许删 → `409 ErrIntegrationVersionLastOne`
- **安装活跃唯一**：`uk_install_instance_tpl_active where deleted_at IS NULL`；卸载是软删（`deleted_at != nil + status='uninstalled'`）
- **reinstall 语义**：再次 `Install` 同 `(instance_id, template_id)` 时，复用旧记录 ID（清掉 `deleted_at`、刷新 status / version / parts），revision `action='reinstall'`
- **租户隔离**：`InstallRequest.TenantID` 必须与目标 instance 的 tenant 一致；非 admin 不能跨租户安装
- **applier**：基于 `integration.Renderer` 渲染 collector / vmrule / dashboard，`CompositeApplier` 派给 k8s + grafana resolver
  - k8s resolver：根据 `cluster_id` 动态构造 `k8s.Client`，并发安全缓存（fingerprint = `incluster|path|inline_hash`）
  - grafana resolver：根据 `host_id` 动态构造 `grafana.Client`，未注册主机时回退平台默认
  - 资源标签：`managed-by=ops-system` / `template=<id>` / `installation=<id>`，Uninstall 时按 label 删
- **dry-run**：`POST /install/plan` 只走渲染 + diff，不落库

### 4.6 指标库

- 解析器 `metric_parser_service` 从 collector spec / dashboard JSON 中提取指标
- 模型唯一约束：`(component, name)` 联合 partial unique；`name` 单独保留普通索引用于模糊搜索
- `manual_override=true` 的条目 reparse 时不被覆盖
- `MetricTemplateMapping` 记录指标在 collector / dashboard / alert 中的出现位置 + dashboard panel

### 4.7 用户 bootstrap

- `POST /users/bootstrap` 完全公共，但仅在系统中"无任何 admin"时允许；已存在则 `409`
- 创建第一个 admin 时同时落审计

## 5. 模型与索引（要点）

| 表 | 主要字段 | 索引 |
|---|---|---|
| `users` | role/department_id/tenant_id | role + tenant_id |
| `instances` | tenant_id/type/spec | (tenant_id, status) |
| `scale_events` | instance_id/scale_type/method/status | (instance_id, created_at desc)、scale_type、method、status |
| `ops_integration_templates` | name/category/component/status | uk_integration_tpl_name_active (where deleted_at IS NULL)、category、component |
| `ops_integration_template_versions` | template_id/version | uk_tpl_version (template_id, version) |
| `ops_integration_installations` | instance_id/template_id/tenant_id/status | uk_install_instance_tpl_active (where deleted_at IS NULL)、tenant_id、status |
| `ops_integration_installation_revisions` | installation_id/action/status | installation_id |
| `ops_metrics` | component/name | uk_metric_component_name、name (非唯一，模糊搜索) |
| `ops_metric_template_mappings` | metric_id/template_id | metric_id、template_id |
| `ops_log_instances` | tenant_id | tenant_id |
| `ops_grafana_hosts` | scope/tenant_id/status | scope、tenant_id |
| `clusters` | name/status | name |
| `platform_scale_audits` | target_id/status/created_at | (target_id, created_at desc)、status、operator |

软删字段统一 `deleted_at gorm.DeletedAt`；审计相关表（`scale_events`、`integration_installation_revisions`、`platform_scale_audits`）**不软删**，保证不可篡改，清理走冷归档。

## 6. 幂等与审计

### 6.1 幂等

- 适用：所有写操作（POST/PUT/DELETE 中的非读语义）
- 头：`Idempotency-Key`（建议 UUID v4）
- 存储：Redis `SET key value NX EX <ttl>`；命中重放返回原响应 + `X-Idempotency-Replayed: true`
- 失败重放：上一次失败也会缓存（避免无限重试），TTL 默认 24h
- Redis 不可用：写操作仍执行，幂等能力降级，日志告警

### 6.2 审计

| 表 | 触发 | 不可变 |
|---|---|---|
| `platform_scale_audits` | 平台扩容 dry-run / apply / replay | ✓ |
| `scale_events` | 实例 scale，含 rejected | ✓ |
| `ops_integration_installation_revisions` | install/upgrade/reinstall/uninstall | ✓ |

## 7. 错误处理与状态码

| HTTP | 典型场景 |
|---|---|
| 400 | 参数错误、UUID 解析失败、必填缺失、`ErrInvalidPagination` |
| 401 | 未登录、token 过期、token 在黑名单、bootstrap 已存在管理员之外用 |
| 403 | 角色不足、跨租户访问（`ErrIntegrationTenantMismatch`） |
| 404 | 资源不存在（`ErrIntegrationTemplateNotFound` 等） |
| 409 | 模板/版本/安装冲突（`ErrIntegrationTemplateNameExists`、`ErrIntegrationTemplateInUse`、`ErrIntegrationVersionExists`、`ErrIntegrationVersionInUse`、`ErrIntegrationVersionLastOne`） |
| 422 | 业务参数语义错误 |
| 429 | 限流（`middleware.RateLimit`） |
| 500 | 未分类异常（响应统一脱敏，详细错误进 zap） |
| 503 | K8s/Helm/Grafana/N9E 不可用 |

handler 通过 `errors.Is` 区分 service 层 sentinel 错误后映射状态码；前端 `extractApiError` 优先消费 message 字段。

## 8. 安全

- 密码哈希：bcrypt，cost 由 `OPS_BCRYPT_COST` 控制（默认 10，生产建议 12+）
- JWT：
  - 密钥来自 `OPS_JWT_SECRET`，长度校验 ≥ 32；启动时强制
  - claims 含 `jti`，logout / 强制下线靠 jti 写黑名单（Redis）
  - 默认有效期来自 `cfg.JWT.ExpireHours`
- 限流：IP 维度（`middleware.NewIPRateLimiter`），可配置
- CORS：`cfg.CORS.AllowedOrigins` 显式白名单
- 凭据落库：`grafana_hosts.admin_token_enc` 走加密列；日志中不打印密钥与 token
