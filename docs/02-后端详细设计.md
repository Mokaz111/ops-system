# 后端详细设计（现网对齐版）

## 1. 技术栈

- Go + Gin
- GORM + PostgreSQL
- Redis（幂等）
- client-go + Helm
- Zap 日志

## 2. 分层结构

- `handler`：HTTP 输入输出与错误映射
- `service`：业务规则与跨组件编排
- `repository`：数据库持久化
- `k8s` / `helm`：基础设施操作封装
- `middleware`：JWT、限流、CORS、请求日志

## 3. 路由与权限

公共接口：

- `POST /api/v1/auth/login`
- `POST /api/v1/users/bootstrap`
- 健康检查：`/health`、`/api/v1/health`、`/api/v1/health/db`

鉴权后接口（JWT）：

- 部门、租户、用户、实例、Grafana、告警集成等查询类接口

管理员接口（RequireRole admin）：

- 部门/租户/用户/实例管理写操作
- 实例扩缩容入口 `POST /api/v1/instances/:id/scale`
- 平台级扩容与审计：
  - `GET /api/v1/platform/scaling/audits`
  - `GET /api/v1/platform/scaling/vmcluster/targets`
  - `POST /api/v1/platform/scaling/vmcluster`

## 4. 关键业务规则

### 4.1 多租户访问隔离

- 非 admin 用户仅允许访问自身租户范围数据
- 用户更新接口限制为“本人可改本人，admin 可改任意用户”

### 4.2 实例扩缩容策略（策略 A）

- `shared` 与 `dedicated_cluster`：实例级扩容直接拒绝
- `dedicated_single`：仅允许垂直扩容与存储扩容

### 4.3 平台级扩容

- 仅支持注册目标（`target_id`）
- 支持 dry-run 与应用
- 通过 K8s CR patch 执行 VMCluster 规格变更

## 5. 幂等与审计

幂等：

- 非 dry-run 请求必须带 `Idempotency-Key`
- 使用 Redis `SET NX + TTL` 存储已完成结果
- 重放返回 `X-Idempotency-Replayed: true`

审计：

- 每次平台扩容请求（成功/失败/重放）写入审计表
- 审计查询支持按时间、目标、状态、操作者筛选

## 6. 错误处理

- 参数错误返回 400
- 未授权或权限不足返回 401/403
- 依赖不可用（如 K8s 操作器不可用）返回 503
- 其他异常统一返回 500
