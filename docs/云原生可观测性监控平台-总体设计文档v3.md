# 云原生可观测性监控平台 - 总体设计文档 V3.0

**文档版本**: V3.0

**创建日期**: 2026-03-28

**核心技术栈**: Golang 1.25.8 + React + Material Design 3 + VictoriaMetrics + 夜萤V8

**设计理念**: 用户友好界面、全自动化部署、无感知底层实现

---

## 一、文档概述

### 1.1 文档目的

本文档为云原生可观测性监控平台的**总体设计规范**，目标是将监控能力封装为平台服务，用户通过 Web 界面即可完成所有操作，无需关注底层 Kubernetes、Helm、VictoriaMetrics 等组件的部署和配置。

### 1.2 核心目标

1. **全界面化运营**: 用户通过 Web 控制台完成租户创建、监控实例开通、告警配置等所有操作
2. **自动化部署**: 后端自动完成 Helm Release 部署、配置更新、资源调度
3. **多租户隔离**: 租户与 VMuser 资源一一对应，实现资源、数据、权限全维度隔离
4. **可观测性统一**: 整合指标(Metrics)+日志(Logs)+告警，一站式解决方案

**关键设计**:

- **1个部门 = 1个租户**: 部门与租户一一对应
- **部门下用户**: 属于该部门的用户自动关联到对应租户
- **权限继承**: 用户权限 = 部门权限 + 租户权限

### 5.2 隔离策略

- **网络隔离**: 每个租户独立 Namespace + NetworkPolicy
- **数据隔离**: VMuser tenant_id 作为写入标识
- **资源隔离**: ResourceQuota 限制 CPU/内存/存储
- **存储隔离**: 独立 PVC，路径含 tenant_id

## 八、总结

### 8.1 设计亮点

1. **全界面化**: 用户无感知底层实现，一切皆可配置
2. **自动化编排**: Golang 后端自动处理 K8s/Helm 操作
3. **Google 设计**: Material Design 3 统一视觉体验
4. **多租户对齐**: 租户与 VMuser 一一对应，简化身份认证

### 8.2 技术价值

- **降低门槛**: 用户无需了解 K8s/Helm/VM 即可使用监控平台
- **提升效率**: 后端自动化编排，分钟级开通实例
- **统一体验**: 前后端标准化开发，易于维护扩展
