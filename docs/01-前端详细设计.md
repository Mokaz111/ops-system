# 前端详细设计（现网对齐版）

## 1. 技术栈

- React 19 + TypeScript
- MUI 7（Material Design 3）
- React Router v6
- Zustand（状态管理）
- axios + 自封装拦截器
- notistack（toast）
- Vite

## 2. 目录结构

```
frontend/src
├── api/            后端 API 封装（axios 实例 + 业务客户端）
│   ├── index.ts          全局 axios 实例 + extractApiError
│   ├── client.ts         AbortController 工具
│   ├── integration.ts    接入中心
│   ├── metric.ts         指标库
```

## 3. 路由与侧边栏

路由元数据集中在 `src/config/appRoutes.ts`，由 `Sidebar` 与 `router.tsx` 共同消费。

### 3.1 Sidebar 分区

| section | 菜单 |
|---|---|
| `overview` | 概览（Dashboard） |
| `resource` | 部门管理 / 租户管理 |
| `monitor` | 监控实例 / 接入中心 / 指标库 |
| `logs` | 日志实例 / 日志查询 |
| `visualization` | Grafana 管理 / Grafana 主机 / Dashboard 管理 |
| `system` | 用户管理* / 集群管理 / 平台扩容* / 系统设置 |

带 `*` 的为 `requireAdmin: true`：路由仍可访问（后端 GET 对所有登录用户开放），但 sidebar 仅在 `user.role === 'admin'` 时显示，避免给 viewer 暴露平台维护入口。

### 3.2 隐藏路由

`alerts`（`/alerts`）保留路由但不出现在 sidebar，仅供 InstanceDetail 内链跳转使用。

## 4. 鉴权与状态

### 4.1 useAuthStore（单一可信源）

- 持久化：`token` / `user` 存 Zustand persist + `localStorage`
- 登录：`POST /auth/login` 成功后写 store；失败用 `extractLoginError` 给出 401/403 → "用户名或密码错误"，其它走 `extractApiError`
- 登出：调 `POST /auth/logout` 把 token 加入黑名单 + 清 store
- 全局 401：axios 拦截器 catch 401 → 清 `localStorage.token` + `window.dispatchEvent(UNAUTHORIZED_EVENT)`；store 模块挂监听同步把 `user/token/isAuthenticated` 置空，`AuthGuard` 立即跳登录

### 4.2 路由守卫

- `AuthGuard`：从 `useAuthStore.token` 读，未登录 → `/login`
- `GuestGuard`：已登录访问 `/login` → `/dashboard`

## 5. 错误处理（extractApiError）

定义于 `api/index.ts`，所有页面 `catch` 必须使用：

```ts
catch (err) {
  enqueueSnackbar(extractApiError(err, '操作失败'), { variant: 'error' });
}
```

提取顺序：

1. 后端 `data.message` / `data.error`（让 409 "模板已被活跃安装引用" 等业务语义直接透出）
2. 状态码兜底中文映射（401/403/404/409/422/429/500）
3. 网络错误（无 response）→ "网络异常，无法连接后端"
4. fallback

效果：被替换的 23 处 catch 不再统一吐 "操作失败"，模板/版本/安装等冲突信息可以直接被运维看到。

## 6. 请求取消（AbortController）

`api/client.ts` 提供：

- `makeAbortController()`
- `isAbortError(err)`：兼容 `axios.isCancel` 与 `AbortError`
- `withSignal(cfg, signal)`

接入了取消的页面：

- `pages/Integration/index.tsx`：分类切换 / 关键字 / `reloadTick` 触发并发拉取（categories / templates / instances / clusters / grafanaHosts）时，cleanup 中 abort 防止旧响应覆盖新结果
- `pages/InstanceDetail/index.tsx`：实例切换时取消上一组（详情 / metrics / scale_events / installations）请求

API 客户端 `list` / `get` 方法签名统一支持可选 `config?: AxiosRequestConfig`，便于传 `signal`。

## 7. 关键页面

### 7.1 Dashboard / 概览
平台概览，强调"避免假数据"的展示原则。

### 7.2 Tenant / Department
对接租户/部门 CRUD，普通用户只能看到自己租户。

### 7.3 Monitor · Instance / InstanceDetail

InstanceDetail 拆为 4 Tab：

| Tab | 内容 |
|---|---|
| 基本信息 | 元数据 / 配额 / 状态 |
| 数据采集 | 该实例上的接入安装列表（状态、版本、已应用资源、最近一次 revision），点击跳接入中心带 `?instanceId=` |
| Dashboard | 该实例已下发的 Grafana dashboard 列表 |
| 告警 | 已下发的 VMRule（只读） + N9E 跳转占位 |

状态/动作文案统一用 `statusLabel` / `actionLabel` 翻译，覆盖 `success/partial/failed/uninstalled/uninstall_failed` 与 `install/upgrade/reinstall/uninstall`。失败 revision 的 Alert severity 按 status 决定。

### 7.4 Monitor · Integration（接入中心）

- 左侧分类树（监控/数据库/中间件/基础设施/日志/云产品） + 顶部搜索
- 卡片列表（component、最新版本、tag）
- 模板抽屉：版本管理（含 AddVersion / DeleteVersion，最后一个版本/被引用版本会被 409 拒绝）、已集成实例、模板信息
- 安装向导：实例选择 → 勾选 parts（collector/vmrule/dashboard） → dry-run diff 预览 → 确认安装
- 删除模板的确认文案明确："会软删所有版本，但活跃安装会被 409 拒绝，需先在对应实例卸载"

### 7.5 Monitor · Metric（指标库）
DataGrid 列出指标，按组件聚合；详情抽屉展示标签、示例 PromQL/MetricsQL、关联模板与 dashboard panel。

### 7.6 Logs · LogInstance / LogQuery
日志实例 CRUD；LogsQL 查询台调 `POST /log-instances/:id/query`。

### 7.7 Visualization · Grafana / GrafanaHost / Dashboard
- Grafana 管理：组织、用户、数据源
- Grafana 主机：注册平台共享 / 租户自带主机
- Dashboard 管理：内置模板与已安装记录

### 7.8 System · PlatformScaling
平台扩容（admin）：

- 注册目标选择
- dry-run 预览 `spec_patch`
- 提交时自动带 `Idempotency-Key`
- 历史审计：目标 / 状态 / 操作者 / 时间范围筛选 + 一键重置 + `spec_patch` 详情

### 7.9 System · Cluster
K8s 集群注册表；接入中心安装时按 `cluster_id` 选择下发目标。

### 7.10 System · Settings
- 个人信息维护
- 平台连接信息展示
- 共享集群初始化（admin）：dry-run + 应用 `vm/victoria-metrics-k8s-stack`

## 8. API 约定

- 所有请求统一走 `api/index.ts` 的 axios 实例
- 请求拦截器：附加 `Authorization: Bearer <token>`（每次重新从 `localStorage.token` 读，确保拿到最新值；写入只走 `useAuthStore`）
- 响应拦截器：401 → 清 token + 广播 `UNAUTHORIZED_EVENT`；网络异常按 `extractApiError` 兜底
- 写操作幂等键由调用方生成（默认 `crypto.randomUUID()`）

## 9. 设计规范

- UI 统一使用 MUI 组件与主题
- 优先简洁可用，避免假数据 / 假完成
- 文案与后端能力边界对齐（如告警引擎独立运行、扩容策略 A 等）
- 不在前端缓存敏感字段，token 只走 store + localStorage 持久化
