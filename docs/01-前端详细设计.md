# 前端详细设计（现网对齐版）

## 1. 技术栈

- React 19 + TypeScript
- MUI 7（Material Design 3）
- React Router
- Zustand
- Axios

## 2. 目录结构（当前）

关键目录：

- `src/pages`：业务页面
- `src/api`：后端 API 封装
- `src/components`：布局与通用组件
- `src/stores`：状态管理
- `src/types`：类型定义

## 3. 页面与路由

路由定义在 `src/router.tsx`，当前页面：

- `/login`
- `/dashboard`
- `/departments`
- `/tenants`
- `/instances`
- `/grafana`
- `/alerts`
- `/users`
- `/settings`
- `/platform-scaling`

认证策略：

- 登录态由本地 `token` 判断
- 未登录跳转 `/login`

## 4. 核心页面说明

### 4.1 Dashboard

- 展示平台概览信息与集成占位说明
- 不再使用误导性的假数据图表

### 4.2 Tenant / Instance

- 对接租户与实例管理接口
- 实例页遵循后端扩缩容策略限制

### 4.3 Alert

- 告警由 N9E 独立运行
- 页面定位为告警集成与查看入口

### 4.4 PlatformScaling

平台扩容页面提供：

- 目标选择（注册目标）
- dry-run 预览与确认应用
- 幂等请求提交
- 变更历史查询（admin）
- 筛选（目标/状态/操作者/时间范围）
- 变更详情查看（`spec_patch`）
- 一键重置筛选

### 4.5 Settings

系统设置页面提供：

- 个人信息维护（邮箱、手机号）
- 平台信息展示（API/Grafana/N9E 地址）
- 共享集群初始化（admin）：支持 `vm/victoria-metrics-k8s-stack` 的 dry-run 与确认应用

## 5. API 约定

- 统一通过 `src/api/index.ts` 的 Axios 实例访问后端
- 共享集群初始化接口：`POST /api/v1/platform/scaling/bootstrap/shared/init`
- 平台扩容提交支持 `Idempotency-Key` 请求头
- 分页响应使用 `{ items, total, page, page_size }`

## 6. 设计规范

- UI 统一使用 MUI 组件与主题
- 优先简洁可用，避免未实现功能的“伪完成”展示
- 页面文案与后端能力边界保持一致
