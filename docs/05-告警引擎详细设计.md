# 告警引擎详细设计

> **架构说明（V3.1 更新）**
>
> 自 V3.1 起，告警引擎（N9E / 夜莺）作为**独立系统**单独部署和运维，
> 不由可观测性平台直接集成管理。本文档描述 N9E 的告警能力设计，
> 供独立部署参考。平台侧聚焦 VM 生命周期管理和 Grafana 可视化管理。
>
> N9E 与平台的对接方式：
> - N9E 通过 n9e-edge 连接各租户的 VictoriaMetrics 实例
> - 告警规则、通知渠道、告警事件均在 N9E 独立界面管理
> - 平台不再维护 alert_rules / alert_events / notification_channels 表

## 3. 告警规则设计

### 3.1 规则结构

```go
// 告警规则结构
type AlertRule struct {
    ID          string    `json:"id"`
    TenantID    string    `json:"tenant_id"`     // 租户 ID
    RuleName    string    `json:"rule_name"`     // 规则名称
    RuleType    string    `json:"rule_type"`     // metrics / logs
    Query       string    `json:"query"`         // PromQL / LogQL
    Condition   Condition `json:"condition"`     // 触发条件
    Level       string    `json:"level"`         // critical / warning / info
    Channels    []string `json:"channels"`      // 通知渠道
    Annotations string    `json:"annotations"`   // 备注
    Enabled     bool      `json:"enabled"`       // 是否启用
    CreatedAt   time.Time `json:"created_at"`
    UpdatedAt   time.Time `json:"updated_at"`
}

type Condition struct {
    Operator   string `json:"operator"`    // >, <, ==, !=, =~
    Threshold  float64 `json:"threshold"`  // 阈值
    For        string `json:"for"`         // 持续时间 (如 5m)
    EvalInterval string `json:"eval_interval"` // 评估间隔
}
```

### 3.2 规则示例

**指标告警规则**:

```yaml
# CPU 使用率告警
- rule_name: "CPU 使用率告警"
  rule_type: "metrics"
  query: "sum(rate(container_cpu_usage_seconds_total[5m])) by (pod) / sum(container_spec_cpu_quota[5m]/container_spec_cpu_period[5m]) * 100"
  condition:
    operator: ">"
    threshold: 80
    for: "5m"
    eval_interval: "30s"
  level: "critical"
  channels: ["dingtalk", "email"]
  annotations: "Pod {{ $labels.pod }} CPU 使用率超过 80%"
```

**日志告警规则**:

```yaml
# 错误日志告警
- rule_name: "错误日志告警"
  rule_type: "logs"
  query: '{app="nginx"} | json | level="error" | rate() > 10'
  condition:
    operator: ">"
    threshold: 10
    for: "1m"
    eval_interval: "30s"
  level: "warning"
  channels: ["dingtalk"]
  annotations: "nginx 错误日志速率过高"
```

### 3.3 预置告警模板

| 模板名称 | 规则 | 级别 | 适用场景 |
|---------|------|------|---------|
| **基础资源告警** | CPU > 80%, 内存 > 85%, 磁盘 > 90% | 严重 | 所有租户 |
| **Pod 告警** | Pod 重启频繁, Pod 不健康 | 警告 | K8s 环境 |
| **业务指标告警** | QPS 下降, 延迟升高 | 警告 | 业务应用 |
| **日志异常告警** | ERROR 日志突增 | 警告 | 所有服务 |
| **自定义** | 用户自定义 | 自定义 | 自定义 |

## 5. 告警收敛

### 5.1 收敛策略

```
告警风暴场景:
┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  10:00 - Pod-1 CPU 告警 ─┐                                          │
│  10:00 - Pod-2 CPU 告警 ─┼─▶ 合并为 1 条 ──▶ 通知: "CPU 告警 (10个Pod)"│
│  10:00 - Pod-3 CPU 告警 ─┤                                          │
│         ...             │                                          │
│  10:00 - Pod-N CPU 告警 ─┘                                          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 5.2 收敛规则

| 收敛维度 | 策略 | 示例 |
|---------|------|------|
| **相同规则** | 合并为一条 | 多 Pod CPU 告警 → 1 条 "CPU 告警 (10个)" |
| **相同实例** | 聚合显示 | 多指标告警 → 1 条 "实例告警" |
| **时间窗口** | 段时间内合并 | 5 分钟内同规则告警合并 |
| **静默期** | 重复告警抑制 | 告警后 1 小时内不重复通知 |

### 5.3 收敛配置

```go
// 收敛配置
type AlertDedupeConfig struct {
    GroupBy       []string `json:"group_by"`       // 按哪些字段分组

    SilencePeriod string   `json:"silence_period"` // 静默期
    MaxCount      int      `json:"max_count"`      // 最大合并数量
}

// 示例
config := AlertDedupeConfig{
    GroupBy:       []string{"rule_name", "tenant_id"},
    GroupInterval: "5m",
    SilencePeriod: "1h",
    MaxCount:      100,
```

## 6. 告警历史与统计

### 6.1 告警事件存储

```sql
-- 告警事件表
CREATE TABLE alert_events (
    id                  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id           VARCHAR(100) NOT NULL,
    rule_id             UUID NOT NULL,
    rule_name           VARCHAR(255) NOT NULL,
    level               VARCHAR(20) NOT NULL,
    status              VARCHAR(20) NOT NULL, -- firing, resolved
    start_time          TIMESTAMP NOT NULL,
    end_time            TIMESTAMP,
    details             JSONB,
    created_at          TIMESTAMP DEFAULT NOW()
);

-- 告警事件索引
CREATE INDEX idx_alert_events_tenant_id ON alert_events(tenant_id);
CREATE INDEX idx_alert_events_rule_id ON alert_events(rule_id);
CREATE INDEX idx_alert_events_start_time ON alert_events(start_time);
```

## 7. 与 N9E 集成

### 7.1 N9E API 调用

```go
type N9EClient struct {
    baseURL   string
    username  string
    password  string
    token     string
}

// 创建告警规则
func (c *N9EClient) CreateAlertRule(rule *AlertRule) error {
    payload := map[string]interface{}{
        "name":        rule.RuleName,
        "note":        rule.Annotations,
        "severity":    toN9ESeverity(rule.Level),
        "prom_for":    rule.Condition.For,
        "prom_ql":     rule.Query,
        "enable":      rule.Enabled,
    }
    
    _, err := c.doRequest("POST", "/api/n9e/rules", payload)
    return err
}

// 获取告警事件
func (c *N9EClient) GetAlertEvents(tenantID string) ([]AlertEvent, error) {
    var result struct {
        Data []AlertEvent `json:"dat"`
    }
    
    _, err := c.doRequest("GET", "/api/n9e/alerts", nil, &result)
    return result.Data, err
}
```
