# 告警引擎详细设计（现网对齐版）

## 1. 当前定位

告警引擎由 N9E 独立部署和运维，`ops-system` 不实现自研告警评估引擎。

平台职责：

- 提供告警规则、事件、通知渠道等集成接口
- 提供前端告警页面用于统一入口与可视化管理

## 2. 集成边界

平台侧已实现：

- 规则查询与管理接口（经后端服务与 N9E 客户端交互）
- 告警事件查询与确认
- 通知渠道查询与管理
- 统计接口（summary/trend/by-level/by-rule）

平台侧不负责：

- 告警计算调度引擎
- 告警通知底层投递系统

## 3. 权限规则

- 告警查询接口：登录用户可访问（受租户隔离规则约束）
- 告警写操作：仅 admin

## 4. 运维要求

- N9E 可用性由独立运维体系保障
- 平台应监控 N9E 接口可达性与调用失败率
- 平台文档与前端文案需明确“告警引擎独立运行”边界
