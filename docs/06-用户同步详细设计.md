# 用户同步详细设计（现网对齐版）

## 1. 目标

确保平台用户、租户与外部系统（Grafana、N9E）的组织关系可关联、可追踪。

## 2. 当前实现范围

已具备：

- 平台内用户、部门、租户模型管理
- Grafana 组织与组织用户管理接口
- N9E 团队与用户相关集成能力（通过后端客户端）

约束：

- 以“平台元数据”为主数据源
- 外部系统状态同步由平台服务触发

## 3. 同步触发点（摘要）

- 租户创建/变更时：关联外部组织标识（如 grafana org、n9e team）
- 用户创建/变更时：按角色和租户范围进行外部系统映射

## 4. 一致性与失败处理

- 外部系统调用失败需记录日志
- 关键标识写入数据库后再对外返回成功
- 管理员可通过管理接口进行补偿操作（如重新同步）

## 5. 安全要求

- 不在日志中打印敏感凭据
- API 凭据通过配置注入，不写入代码
- 用户权限变更应立即影响管理接口访问边界
