# 运维监控设计（现网对齐版）

## 1. 监控目标

- 平台 API 服务稳定性
- 数据库与 Redis 可用性
- 外部依赖（K8s、Grafana、N9E）调用健康度
- 平台级扩容链路可追踪性

## 2. 基础健康检查

- 进程健康：`/health`
- API 健康：`/api/v1/health`
- DB 健康：`/api/v1/health/db`

## 3. 关键观测指标建议

- HTTP 请求量、错误率、时延分位
- 数据库连接池与慢查询
- Redis 连接可用率（幂等链路）
- 平台扩容成功率、失败率、重放率

## 4. 审计与排障

平台扩容审计表提供：

- 操作者与来源 IP
- 目标与状态
- 变更 patch 与错误信息
- 时间序列查询能力

可用于：

- 变更追责
- 回溯故障窗口内的配置操作

## 5. 告警建议

- DB 健康检查失败
- Redis 不可用持续超阈值
- 平台扩容失败率短时升高
- 对外系统（Grafana/N9E）调用失败率升高
