## 2. 平台自身监控

### 2.1 监控指标采集

| 组件 | 采集方式 | 指标 |
|------|---------|------|
| API Server | prometheus/client_golang | 请求QPS, 延迟, 错误率 |
| PostgreSQL | postgres_exporter | 连接数, 查询延迟, 复制延迟 |
| Redis | redis_exporter | 内存使用, 命中率, 命令执行 |
| N9E | N9E 内置 | 规则评估, 告警事件, 通知发送 |

### 2.3 告警规则

```yaml
# 平台自身告警规则
- name: "API Server 告警"
  rules:
    - alert: "API响应超时"
      expr: "histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[5m])) > 5"
      severity: "critical"
    - alert: "API错误率高"
      expr: "rate(http_requests_total{status=~"5.."}[5m]) > 0.01"
      severity: "warning"

- name: "数据库告警"
  rules:
    - alert: "PostgreSQL连接池满"
      expr: "pg_stat_activity_count / pg_settings_max_connections > 0.9"
      severity: "critical"
    - alert: "PostgreSQL复制延迟"
      expr: "pg_replication_lag > 10"
      severity: "warning"

- name: "Redis告警"
  rules:
    - alert: "Redis内存使用率高"
      expr: "redis_memory_used_bytes / redis_memory_max_bytes > 0.9"
      severity: "warning"
```

## 5. 备份与恢复

### 5.1 备份策略

```yaml
# 备份配置
backup:
  # PostgreSQL 备份
  postgresql:
    schedule: "0 2 * * *"  # 每天凌晨2点
    retention: "7d"
    method: "pg_dump"
    destination: "s3://backup-platform/postgresql"
  
  # Redis 备份
  redis:
    schedule: "0 * * * *"  # 每小时
    retention: "24h"
    method: "rdb"
    destination: "s3://backup-platform/redis"
  
  # K8s 资源备份
  kubernetes:
    schedule: "0 3 * * *"  # 每天凌晨3点
    retention: "30d"
    method: "velero"
    destination: "s3://backup-platform/k8s"
  
  # VM 数据备份
  victoriametrics:
    schedule: "0 4 * * *"  # 每天凌晨4点
    retention: "7d"
    method: "vmbackup"
    destination: "s3://backup-platform/vm"
```

### 5.2 恢复流程

```bash
# PostgreSQL 恢复
pg_restore -h postgresql.platform -U postgres -d monitoring < backup_20260328.sql

# Redis 恢复
redis-cli -h redis.platform -p 6379 < dump.rdb

# K8s 资源恢复
velero restore restore-20260328 --from-backup backup-20260328

# VM 数据恢复
vmbackup -storageDataPath=/data -s3bucket=s3://backup-platform/vm -restore
```

## 6. 运维工具

### 6.1 运维脚本

```bash
#!/bin/bash
# 租户资源查询脚本

# 查看租户资源使用
function get_tenant_resources() {
    local tenant_id=$1
    
    echo "=== 租户资源使用 ==="
    echo "Tenant: $tenant_id"
    
    # CPU 使用
    kubectl top pods -n tenant-$tenant_id 2>/dev/null | tail -n +2
    
    # 存储使用
    kubectl get pvc -n tenant-$tenant_id -o wide
    
    # VM 指标
    curl -s "http://vm-$tenant_id-vmselect:8481/api/v1/query?query=vm_rows" | jq
}

# 租户实例健康检查
function check_tenant_health() {
    local tenant_id=$1
    
    echo "=== 租户实例健康检查 ==="
    
    # 检查 VM
    kubectl get pods -n tenant-$tenant_id -l app.kubernetes.io/name=victoria-metrics
    
    # 检查日志
    kubectl logs -n tenant-$tenant_id -l app.kubernetes.io/name=victoria-metrics --tail=50 | grep -i error
    
    # 检查告警
    curl -s "http://n9e.platform:18000/api/n9e/alerts?tenant=$tenant_id" | jq
}
```

## 7. 故障处理

### 7.1 故障分类

| 级别 | 定义 | 响应时间 | 恢复时间 |
|------|------|---------|---------|
| P0 | 全平台不可用 | 15 分钟 | 1 小时 |
| P1 | 核心功能不可用 | 30 分钟 | 4 小时 |
| P2 | 非核心功能异常 | 2 小时 | 24 小时 |
| P3 | 轻微问题 | 24 小时 | 72 小时 |

### 7.2 故障处理流程

```
故障检测
    │
    ▼
┌──────────────────┐
│  自动告警通知   │
│  (电话/短信/邮件)│
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  故障定位       │
│  (日志/监控/排查)│
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  故障处理       │

└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  故障恢复       │

         │
         ▼
┌──────────────────┐

```

### 7.3 常见故障处理

| 故障场景 | 症状 | 处理措施 |
|---------|------|---------|
| Pod 反复重启 | CrashLoopBackOff | 查看日志、修复配置 |
| 存储空间不足 | DiskPressure | 扩容 PVC、清理数据 |
| 网络不通 | Connection Timeout | 检查 NetworkPolicy、DNS |
| 认证失败 | 401 Unauthorized | 检查 Token、API Key |
| 写入失败 | 500 Error | 检查 VM 状态、配额 |

## 8. 巡检机制

### 8.1 日常巡检

```yaml
# 每日巡检任务
daily_check:
  time: "09:00"
  tasks:
    - name: "平台状态检查"
      command: "kubectl get pods -n platform | grep -v Running"
      alert_if: "有非 Running Pod"
    
    - name: "存储检查"
      command: "df -h | grep -E '/data|/var/lib'"
      alert_if: "使用率 > 85%"
    
    - name: "告警检查"
      command: "curl -s 'http://n9e.platform:18000/api/n9e/alerts?status=firing' | jq '.dat | length'"
      alert_if: "告警数 > 10"
    
    - name: "备份检查"
      command: "ls -la /backup/* | tail -1"
      alert_if: "备份文件不存在或过旧"
```
